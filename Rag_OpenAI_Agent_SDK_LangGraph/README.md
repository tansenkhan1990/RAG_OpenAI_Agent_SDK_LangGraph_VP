# RAG + OpenAI Agent SDK + LangGraph

This project is a powerful, local-first multi-agent retrieval-augmented generation (RAG) system. It combines the structured workflow orchestration of **LangGraph**, the agent abstractions of the **OpenAI Agents SDK**, and the fast vector similarity search of **Supabase** (powered by `pgvector`). All language models and embeddings run entirely locally using **Ollama**, making it private and cost-effective.

## 🏗️ Architecture & Features

- **FastAPI**: Serves the application as a robust REST API.
- **LangGraph**: Orchestrates the multi-step agent workflow. It manages the application state and dynamically routes queries to the correct specialized agent.
- **OpenAI Agents SDK**: Provides the framework to build individual agents (Router, RAG, Web Search, General) equipped with tools.
- **Supabase (pgvector)**: Acts as the vector database to store document chunks and perform fast similarity searches.
- **Ollama**: Local AI infrastructure powering both the LLM and the embedding models.
- **DuckDuckGo API**: Provides real-time web search capabilities to the web agent.

### Multi-Agent Workflow
1. **Router Agent**: Evaluates the incoming query and decides if it needs private data (`rag`), internet data (`search`), or general knowledge (`general`).
2. **RAG Agent**: Uses the `rag_tool` to convert the query into a vector, searches Supabase, and answers based strictly on retrieved documents.
3. **Search Agent**: Uses the `search_tool` (DuckDuckGo) to fetch real-time data from the web.
4. **General Agent**: Answers basic questions directly using the LLM.

## 📚 Libraries Used

- `fastapi` & `uvicorn`: For building and serving the API.
- `langgraph`: For stateful workflow orchestration.
- `openai` & `openai-agents`: For standardizing communication with the local Ollama API and building agent toolings.
- `supabase`: Python client to interact with the Supabase PostgreSQL database.
- `pypdf`: For extracting text from PDF files during data ingestion.
- `requests`: For making external API calls (e.g., DuckDuckGo search).
- `python-dotenv`: For managing environment variables.

## ⚙️ Prerequisites

1. **Python 3.12+** and [uv](https://github.com/astral-sh/uv) package manager installed.
2. **Ollama** installed locally.
3. A free **Supabase** account.

Pull the required Ollama models:
```bash
ollama pull qwen3-vl:235b-cloud       # Or your preferred local model
ollama pull nomic-embed-text:latest   # Required for embeddings
```

## 🚀 Setup & Installation

### 1. Clone & Install Dependencies
Since the project is managed with `uv`, you can install dependencies quickly:
```bash
uv sync
```
*(Or simply let `uv run` handle dependencies automatically based on `pyproject.toml`)*

### 2. Configure Environment Variables
Create a `.env` file in the root directory (you can use `.env.example` if available) and add the following:
```env
OPENAI_BASE_URL=http://localhost:11434/v1
OPENAI_API_KEY=ollama

# Disable telemetry to prevent the OpenAI SDK from attempting to send traces to the cloud
OPENAI_AGENTS_DISABLE_TRACING=1
OPENAI_DISABLE_TELEMETRY=true

LOCAL_MODEL_NAME=qwen3-vl:235b-cloud
LOCAL_EMBEDDING_MODEL=nomic-embed-text:latest

SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_KEY=your-supabase-anon-key
```

### 3. Database Setup (Supabase)
You need to create the required tables and functions in Supabase:
1. Go to your [Supabase Dashboard](https://supabase.com/dashboard).
2. Open your project and click on the **SQL Editor**.
3. Copy the entire contents of the local `schema.sql` file.
4. Paste it into the editor and click **Run**.

*Note: The schema is pre-configured to use `vector(768)` which matches the output dimension of `nomic-embed-text`.*

### 4. Ingest Documents
To load data into your RAG pipeline, place your PDF files in `src/data/` and run the ingestion script. By default, it looks for `Tansen_Khan_CV.pdf`:

```bash
uv run src/scripts/ingest_pdf.py
```
This script will parse the PDF, chunk the text, compute embeddings via Ollama, and insert them into Supabase.

## 🏃‍♂️ Running the Project

Start the FastAPI development server:
```bash
uv run fastapi dev main.py
```
*(Alternatively: `uv run uvicorn main:app --reload`)*

### Testing the API
You can test the application by sending a POST request to the `/ask` endpoint:

```bash
curl -X POST http://127.0.0.1:8000/ask \
     -H "Content-Type: application/json" \
     -d '{"query": "What is the experience listed in the CV?"}'
```

The response will include the query, the chosen route (e.g., `rag`, `websearch`, `llm`), the source agent, and the final answer generated by the local model.

## 📂 Project Structure

```
├── main.py                  # FastAPI application entry point
├── schema.sql               # Supabase database schema and RPC functions
├── pyproject.toml           # Python dependencies and project config
├── .env                     # Environment variables
├── src/
│   ├── config.py            # Loads configuration from .env
│   ├── agents/
│   │   └── factory.py       # Defines Router, RAG, Web, and General agents
│   ├── database/
│   │   └── supabase_client.py # Supabase client initialization
│   ├── graph/
│   │   ├── state.py         # Defines the LangGraph state schema (TypedDict)
│   │   └── workflow.py      # Connects agents into a directed LangGraph workflow
│   ├── scripts/
│   │   └── ingest_pdf.py    # Standalone script to chunk and embed PDFs
│   └── tools/
│       ├── embedding.py     # Wrapper to generate embeddings via Ollama
│       ├── rag_tool.py      # Performs vector search against Supabase
│       └── search_tool.py   # Performs web search using DuckDuckGo
```
